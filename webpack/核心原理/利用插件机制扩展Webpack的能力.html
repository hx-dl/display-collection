# 利用插件机制扩展 Webpack 的能力

Webpack 的插件机制就是我们在软件开发中最常见的钩子机制。

钩子机制也特别容易理解，它有点类似于 Web 中的事件。在 Webpack 整个工作过程会有很多环节，为了便于插件的扩展，Webpack 几乎在每一个环节都埋下了一个钩子。这样我们在开发插件的时候，通过往这些不同节点上挂载不同的任务，就可以轻松扩展 Webpack 的能力。

# 写一个打包后生成版本发布信息的插件

buildInfo-webpack-plugin

class BuildInfoWebpackPlugin {
  constructor(options) {
    console.log("插件被使用了");
  }
  apply(compiler) {
    //emit表示在文件生成资源到 output 目录之前。  异步时刻
    compiler.hooks.emit.tapAsync("BuildInfoWebpackPlugin", (compilation, cb) => {
      console.log(compilation.assets);
      compilation.assets["buildInfo.txt"] = {
        source: function() {
          return `项目名称: Rainbow\n版本发布时间： ${new Date().toLocaleString()}\nt版本号: v.0.2.0.0.0`;
        },
        size: function() {
          return 1000;
        },
      };
      cb();
    });
  }
}

module.exports = BuildInfoWebpackPlugin;

官方钩子 API 涵盖了 Webpack 整个运行生命周期中的几乎所有的钩子，通过查阅官方文档，我们可以轻松的在需要的时刻完成一些事情并且毫无侵略性。

webpack.config.js

const CopyrightWebpackPlugin = require("./plugins/copyright-webpack-plugin");

/** @type { import('webpack').Configuration } */
module.exports = {
  mode: "none",
  entry: "./src/index.js",
  output: {
    filename: "bundle.js",
  },
  plugins: [new CopyrightWebpackPlugin()],
};

现在大家应该知道为什么要们在使用插件的使用要 new 调用了吧。配置完成之后，我们执行打包，dist目录下生成了一个buildInfo.txt文件，执行成功。

# Plugin机制总结

Plugin通常都是一个类或者构造函数，内部必须实现一个apply方法供Webpack在特定的时机调用。Plugin 侧重用来解决项目中除了资源模块打包以外的其他自动化工作，Webpack 为每一个工作环节都预留了合适的钩子，我们在扩展时只需要找到合适的时机去做合适的事情就可以了。
