# Webpack 如何实现非 JS 文件的加载

Webpack 实现的是整个前端项目的模块化，也就是说，我们可以通过 Webpack 去管理前端项目中任意类型（包括 CSS 文件、图片等）的资源文件。而做到这一点，其实都是依靠 Webpack 的Loader机制。

# Webpack处理特殊模块的规则

我们还是举例子，先来使用 Webpack 对一个 Demo 做打包，不一样的是，我们这次来对一个 css 文件做打包，看看会发生什么？

 └─ Webpack-Css ························ sample root dir
    ├── src ·································· source dir
+   │   └── index.css ························· main styles
    ├── package.json ························· package file
    └── webpack.config.js ···················· webpack config file

/* ./src/index.css */
body {
  margin: 0 auto;
  padding: 0 20px;
  font-size: 20px;
  background: #7ea9ca;
}

然后将 Webpack 配置中的入口文件路径指定为 index.css 的文件路径，让 Webpack 直接打包 CSS 资源文件，具体配置如下所示：

module.exports = {
  // 样式文件路径
  entry: "./src/main.css",
  output: {
    filename: "bundle.js",
  },
};

接着，走你～ 报错了

因为 Webpack 的内部，默认只能处理 JavaScript 代码，所以在没有制定对应的 Loader 的时候，所有的文件都会被当作 JavaScript 进行解析，但是 CSS 代码不符合 JavaScript 语法，所以就报错了。

为了验证这一点，我们可以尝试在 css 文件中写入 JavaScript 代码。打包结果是正常通过的

我们引入以下 css-loader，来处理以下 css 文件,看看打包结果是什么:

// Module
___CSS_LOADER_EXPORT___.push([module.i, "body {\n  margin: 0 auto;\n  padding: 0 20px;\n  font-size: 20px;\n  background: #7ea9ca;\n}\n", ""]);

css-loader将所有的css代码保存在了一个数组里，css模块转换为成了一个js模块，如果在开发中，仅仅使用css-loader是不够的，我们一般还会通过style-loader 将这些css代码插入到html的<style>标签中,如果想要打包出单独的css文件，我们还需要使用mini-css-extract-plugin这个插件将css代码从js模块中抽离出去。

总的来说，Webpack加载非JS模块的做法就是将模块丢给文件对应的Loader进行处理

# Webpack处理图片的原理

对于图片等文件的处理，通常是通过file-loader或者url-loader来实现，其实原理也很简单，这里简单的说一下：

  • 通过引入的路径找到图片文件，直接将图片文件拷贝到loader中配置的最终要导出的目录，并生成一个JS模块记录最终的文件路径导出
  • 如果是部分很小的图片(通常默认配置小于8kb)，url-loader采用转换为base64，注入到js代码中
