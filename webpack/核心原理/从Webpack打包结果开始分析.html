# 从 Webpack 打包结果开始分析

我们通过一些实际的例子，来分析一下Webpack是怎么实现上文提到的 3个功能。

# 使用 Webpack 对项目打包

Webpack 是如何把模块合并到一起，并且使得最后生成的代码还能正常在浏览器工作的? 我们通过使用 Webpack 对下面这个简单的项目进行一次打包，来分析一下 Webpack 打包生成的结果：

└─ Webpack-Example
   ├── src
   │   ├── index.js
   │   ├── module-a.js
   │   └── module-b.js
   └── webpack.config.js

index.js：

import text from "./module-a.js";
console.log(text);

module-a.js

import str from "./module-b.js";
const text = `WXBC ${str}`;
export default text;

module-b.js

export default "行业应用大前端";

webpack.config.js

/** @type { import('webpack').Configuration } */
module.exports = {
  mode: "none",
  entry: "./src/index.js",
  module: {
    rules: [
      {
        test: /\.js$/,
        use: {
          loader: "babel-loader",
          options: {
            presets: ["@babel/preset-env"],
          },
        },
      },
    ],
  },
  output: {
    filename: "bundle.js",
  },
};

这里为了更好的理解打包后的代码，这里将 Webpack 工作模式设置为 none，这样 Webpack 就会按照最原始的状态进行打包，所得到的结果更容易理解和阅读。

小技巧：为 Webpack 配置文件增加智能提示

打包结束后，我们在dist目录下，得到来一个bundle.js文件，我们先将无用的注视及不需要的代码干掉，得到如下结果：

(function(modules) {
  // webpackBootstrap
  // 创建一个对象用于缓存加载过的模块
  var installedModules = {};
  // 实现了一个require函数
  function __webpack_require__(moduleId) {
    // 如果模块已经被缓存，直接返回模块的exports对象
    if (installedModules[moduleId]) {
      return installedModules[moduleId].exports;
    }
    // 没有加载过的模块，就创建一个新的module对象并将id添加到缓存中
    var module = (installedModules[moduleId] = {
      i: moduleId,
      l: false,
      exports: {},
    });
    // 执行模块的代码，往exports对象上挂载内容
    modules[moduleId].call(module.exports, module, module.exports, __webpack_require__);
    // 标志模块加载完成
    module.l = true;
    // 返回exports对象，意味着导出模块内容
    return module.exports;
  }
  // 加载入口模块
  return __webpack_require__((__webpack_require__.s = 0));
})([
  /* 0 */
  function(module, __webpack_exports__, __webpack_require__) {
    "use strict";
    /* harmony import */ var _module_a_js__WEBPACK_IMPORTED_MODULE_0__ = __webpack_require__(1);
    console.log(_module_a_js__WEBPACK_IMPORTED_MODULE_0__["default"]);
  },
  /* 1 */
  function(module, __webpack_exports__, __webpack_require__) {
    "use strict";
    /* harmony import */ var _module_b_js__WEBPACK_IMPORTED_MODULE_0__ = __webpack_require__(2);
    var text = "WXBC ".concat(_module_b_js__WEBPACK_IMPORTED_MODULE_0__["default"]);
    /* harmony default export */ __webpack_exports__["default"] = text;
  },
  /* 2 */
  function(module, __webpack_exports__, __webpack_require__) {
    "use strict";
    /* harmony default export */ __webpack_exports__["default"] = "行业应用大前端";
  },
]);

# 分析 Webpack 的打包结果

整体生成的代码其实就是一个立即执行函数，它接收一个 modules 参数，调用时传入了所有打包后模块的列表。

数组里面的元素均是参数列表相同的函数。这里的函数对应的就是我们源代码中的模块，也就是说每个模块最终被包裹到了这样一个函数中，从而实现模块私有作用域。

工作入口函数内部也非常简单，最开始定义了一个 installedModules 对象用于存放或者缓存加载过的模块。紧接着实现了一个 _webpack_require 函数用来加载模块。最后传入入口模块的 ID 调用__webpack_require__ 函数
